# v1 UI 交互设计

网页整体应该分为两个功能模块：作业登记薄（对应 homework_batch）、今日作业(homework_items)。

首页就是今日作业页面

## 作业登记薄

基本页面：

- 首页按照时间倒序显示最近的7个 homework_batch，第八个位置放置一个「+」按钮，新增作业。
- 每个 homework_batch 采用书本的样式显示.
- 书本布局设计需要考虑响应式布局，我们会在 PC、移动端、pad 端使用。
- 每个书本显示摘要信息，标题使用批次名，作业完成进度，截止时间、状态徽章
- homwork_batch 数据为空时，需要展示一个默认的「+」卡片，提示：“还没有作业，点击添加作业”
- 每个书本中应该有一个「编辑作业」按钮，PC 侧通过 hover 展示，移动端通过长按触发
- 每个书本卡片点击的默认动作是，进入「作业详情」的页面。

书本布局核心规则

  | 规则      | 说明                                          |
  |-----------|-----------------------------------------------|
  | 布局方式  | CSS Grid，「+」作为网格内的一个单元格         |
  | 初始加载  | 填满一屏减一格，最后一格放「+」               |
  | 滚动加载  | 每次加载一栏数量的数据，「+」被推到下一行末尾 |
  | 「+」位置 | 始终在当前数据的最后一行末尾                  |

  响应式参数

  | 屏幕尺寸          | 一栏数量 | 初始加载 | 滚动加载 |
  |-------------------|----------|----------|----------|
  | PC (≥1024px)      | 4        | 7 个     | 4 个     |
  | 平板 (768-1023px) | 3        | 5 个     | 3 个     |
  | 移动 (<768px)     | 2        | 3 个     | 2 个     |

「新建作业」页面：

- 点击「+」进入页面
- 页面分为 3 块：顶部是图片列表、中间是已有的homework_batch 列表，deadline_at、底部是「保存」按钮。
- 图片列表展示缩略图、点击展开大图，大图之间可左右预览
- 图片列表最右侧放置「+」，上传新图片。注意：布局自适应处理，「+」按钮始终展示出来。
- 点击图片列表「+」，展示弹窗，批量上传图片文件，后端解析完成后，关闭弹窗，渲染页面对应内容。
- 点击「保存」按钮，回到“作业登记簿”首页。

「编辑作业」 页面：
编辑作业页面和「新建作业」页面保持一致。只是它所有的操作都是在已有的 homework_batch 上进行修改。

## 作业详情

- “作业登记簿”点击「进入作业」
- 根据 作业的状态，展示不同的页面
- 分为两个部分：顶部是关联的所有图片大图列表，这里图片占大部分空间；下面是作业列表
- 顶部图片平铺展示，如果数量过多则左右滑动展示，“homework”类型图片在最左侧。
- 底部作业列表展示：subject、内容、状态、「完成」按钮
- 已完成的作业在最下面
- 今日作业的最顶部，需要展示 deadline_at 提示信息：「距离截止时间还有 XX 小时 XX 分钟」，如果已经超过截止时间，则展示「已超过截止时间 XX 小时 XX 分钟」
